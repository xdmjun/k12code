# 组件复用

同一个页面内多个操作复用UI相同的组件

  • 如存在非幂等操作建议使用多个model对应
  • 幂等操作(get)可以使用同一个变量去显示

例:A、B、C三种操作共用同一个list(model)，A对list的修改会影响B操作时的数据一致性

# 请求顺序

页面初始化时多个请求存在依赖顺序时要确保按顺序请求，不能并列请求

# 组件复用保证组件唯一性

复用多个name相同的组件或者利用ref取dom节点时，建议加上key或者使用不同的ref的名称，防止vue更新组件状态时找错节点

# js对象取属性值，打印这个对象可以看到这个属性，但是取不到

// user = { user_type: 1 }
let user = state.user;    
console.log(user);
console.log(user.user_type);

user对象是可以打印出来的，但是取user_type取到的却是undefined。

# 响应式属性

页面上v-mode需要绑定对象内的属性时，例如弹出浮层页面内的表单属性需要设置默认值的情况应尽量避免对属性直接赋值(非响应式)，使用$set方法去赋值

# 文件导入操作

  • 需要注意是否设置了合适的超时时间
  • 导入涉及的所有数据变化是否同步最新状态

# $refs

this.$refs调用元素属性时注意判断元素是否存在(undefined 时会影响后续代码执行)

# for循环用法总结

性能

  • for 循环最简单，没有任何额外的函数调用栈和上下文；
  • for...of只要具有Iterator接口的数据结构，都可以使用它迭代成员。它直接读取的是键值。
  • forEach，因为它其实比我们想象得要复杂一些，它实际上是array.forEach(function(currentValue, index, arr), thisValue)它不是普通的 for 循环的语法糖，还有诸多参数和上下文需要在执行的时候考虑进来，这里可能拖慢性能；
  • map() 最慢，因为它的返回值是一个等长的全新的数组，数组创建和赋值产生的性能开销很大。
  • for...in需要穷举对象的所有属性，包括自定义的添加的属性也能遍历到。且for...in的key是String类型，有转换过程，开销比较大

适用场景

  • 需要将数组按照某种规则映射为另一个数组，用 map。
  • 需要进行简单的遍历，用 forEach 或 for of。
  • 需要对迭代器进行遍历，用 for of。
  • 需要过滤出符合条件的项，用 filterr。
  • 需要先按照规则映射为新数组，再根据条件过滤，用map加filter

# element 抽屉打开的默认操作

抽屉打开时会查找抽屉里第一个元素进行focus,可能会影响某些操作
node_modules/element-ui/packages/drawer/src/main.vue L127~L148
临时处理办法: 在抽屉内加一个高度为0的输入框

watch: {
    visible(val) {
      if (val) {
        this.closed = false;
        this.$emit('open');
        if (this.appendToBody) {
          document.body.appendChild(this.$el);
        }
        this.prevActiveElement = document.activeElement;
        this.$nextTick(() => {
          Utils.focusFirstDescendant(this.$refs.drawer);
        });
      } else {
        if (!this.closed) this.$emit('close');
        this.$nextTick(() => {
          if (this.prevActiveElement) {
            this.prevActiveElement.focus();
          }
        });
      }
    }
},
